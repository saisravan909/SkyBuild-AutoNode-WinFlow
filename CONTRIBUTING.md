# Contributing to SkyBuild: AutoNode WinFlow

First off, thank you for considering contributing to SkyBuild! It’s people like you that make the Atlassian community such a vibrant place to innovate.

## How Can I Contribute?

###  Technical Contributions
We are currently looking for help with **Phase 2** of our roadmap:
* **Security Integration:** Helping bridge SkyBuild with Atlassian Guard.
* **Secret Management:** Implementing secure credential handling (HashiCorp Vault or GitHub Secrets).
* **Provider Expansion:** Adding support for VMware or Azure-native provisioning logic.

###  Documentation & Feedback
* **Bug Reports:** If you find a glitch in the WinFlow engine, please open an Issue.
* **Feature Requests:** Have an idea to make the JSM Assets schema better? Let us know!
* **Case Studies:** If you’ve implemented SkyBuild in your lab or production, we’d love to hear your story.

## My Philosophy
SkyBuild is built on the principle of **Passion for the Community**. When contributing, please ensure your code is:
1. **Readable:** Clear comments and PowerShell best practices.
2. **Governed:** Every automation should point back to a Jira Asset for auditability.
3. **Inclusive:** We welcome contributors of all skill levels.

## Your Recognition
All contributors will be featured in the "Community Heroes" section of the main README.
